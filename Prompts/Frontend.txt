========================================================
PROJECT OVERVIEW
================
Project Name: LILA BLACK Analytics
Purpose: A single-page dashboard allowing Level Designers to view player data by simply choosing a Date from a single dropdown menu.

The application connects to a backend server via WebSockets. When a user selects a date (e.g., "February_10"), the backend automatically streams the available maps, match statistics, event points, and pre-computed heatmaps.

========================================================

3. Replay Timeline: Simple playback controls (Play, Pause, Progress Slider) to playback match events over time based on timestamps.
4. Heatmap Toggle: A checkbox/switch to overlay a transparent heatmap image with an opacity slider.

========================================================
WEBSOCKET PROTOCOL & INCOMING DATA FLOW
=======================================
On dropdown change, the app sends a single JSON message over WebSockets:
{
  "action": "fetch_date_data",
  "date": "February_10"
}

The app must catch and handle four distinct incoming JSON message packets:
1. "init_data": Updates the available dates array and sets the initial maps list.
2. "match_stats": Contains a list of match performance metrics to display on the dashboard panel.
3. "events_stream": Contains chronological gameplay coordinates sorted by timestamp used to drive the telemetry path replay.
4. "heatmap_data": Provides base64-encoded PNG image strings ("data:image/png;base64,...") for traffic, kills, deaths, and loot.
